# SmartSleepAlarm 真机复核操作手册（阶段5）

> 版本：MVP V1.0  
> 日期：2026-03-30  
> 适用范围：iPhone + Apple Watch 真机联调封板

## 1. 复核目标

1. 功耗：watch 监测期间平均耗电 `< 5%/小时`。  
2. 长时稳定性：连续运行无崩溃、无卡死、关键链路不断。  
3. 异常场景：低电量、断连、未佩戴、权限缺失等场景下不漏叫。  
4. 验收：补齐需求文档 9 条验收项的真机证据（录屏+日志+记录表）。

## 2. 测试环境准备

1. 设备：
- iPhone（iOS 17+）
- Apple Watch（watchOS 10+）
- 两者已配对、同一 Apple ID

2. 构建与安装：
- 使用 Xcode Debug 真机安装 iOS App + watch App + watch Extension
- 确认两端版本一致（同一 commit）

3. 权限：
- 通知：允许
- HealthKit：允许心率读取（和项目要求的写入项）

4. 时间设置：
- iPhone 与 watch 自动设置时间（避免时间漂移）

5. 日志导出路径（应用容器）：
- `Library/Application Support/SmartSleepAlarm/Logs/runtime-events.jsonl`

## 3. 通用执行流程（每条用例都遵循）

1. iOS 新建闹铃，设置为“当前时间 + 35 分钟”（覆盖 `-30 分钟预热`）。  
2. 开启智能模式，设置贪睡分钟（例如 5 或 7）。  
3. 启动录屏：
- iPhone 屏幕录制
- watch 屏幕录制（或外部拍摄）
4. 观察关键节点：
- `T-30min`：是否预热
- `T0`：是否响铃
- 清醒后：是否在 5 秒内静音
- 静音后 5 分钟：再睡是否重响
5. 用例结束后导出：
- iOS / watch 容器日志 `runtime-events.jsonl`
- 失败场景补充系统日志
6. 填写执行表（见 `docs/真机复核执行表模板.md`）。

## 4. 功耗复核

## PWR-01 预热监测功耗（1小时）
1. watch 电量记录起始值（建议 >80%）。  
2. 运行 60 分钟，仅保留本应用监测链路。  
3. 记录结束电量。  
4. 通过标准：耗电 `<= 5%/小时`。

## PWR-02 完整链路功耗（3轮）
1. 连续跑 3 次“响铃-静音-防再睡”流程。  
2. 每轮记录起止电量与时长。  
3. 通过标准：平均耗电速率 `<= 5%/小时`，且无漏叫。

## PWR-03 夜间长测（6-8小时）
1. 夜间持续运行。  
2. 至少触发一次完整闹铃流程。  
3. 通过标准：无崩溃、无异常退出、关键节点可追溯。

## 5. 长时稳定性复核

## STB-01 多轮状态机稳定性
1. 连续 10 轮短间隔验证。  
2. 预期状态迁移可观察：
- `prewarming -> ringing -> silenced/monitoring -> reringing`
3. 通过标准：10 轮全通过。

## STB-02 后台会话稳定性
1. 预热后锁屏，保持后台。  
2. 到点验证仍可响铃并进入后续流程。  
3. 通过标准：不中断；若中断需降级且不漏叫。

## STB-03 重启恢复
1. 预热阶段分别重启 iPhone / watch。  
2. 验证恢复后仍不漏叫。  
3. 通过标准：闹铃不丢、流程可恢复。

## 6. 异常场景复核

## EXC-01 低电量
1. watch 电量降到 <20%，再测试 <10%。  
2. 预期：降级或保底响铃，不漏叫。  
3. 日志关注：`lowBattery` 或等效降级原因。

## EXC-02 断连
1. 预热前/预热后分别断连：
- iPhone 关蓝牙
- watch 飞行模式
- 拉远距离
2. 预期：断连仍不漏叫，恢复后可继续同步。  
3. 日志关注：`connectivityFailure` / 排队重试。

## EXC-03 未佩戴
1. 响铃前摘下手表。  
2. 预期：识别未佩戴并降级，不漏叫。  
3. 日志关注：`watchNotWorn`。

## EXC-04 权限缺失（Health）
1. 关闭 Health 权限后触发闹铃。  
2. 预期：降级保底响铃。  
3. 日志关注：`healthPermissionMissing`。

## EXC-05 传感器超时
1. 制造采样缺失场景（静置+权限变化等）。  
2. 预期：进入 `sensorTimeout` 降级且不漏叫。  

## 7. 9条验收项与证据映射

1. 创建/编辑/删除闹铃：录屏 + 执行表 + JSONL。  
2. 提前30分钟预热：`prewarm_sent/prewarm_received` 日志时间戳。  
3. 清醒后5秒内静音：`ring -> silenced` 时间差。  
4. 5分钟内再睡重响：`monitoring -> reringing`。  
5. 手势贪睡后X分钟重响：`snooze_event -> snoozeTimeout -> reringing`。  
6. 贪睡期间再睡仍重响：同上链路循环证据。  
7. 传感器缺失不漏叫：`degraded` + 保底响铃录屏。  
8. 详情页开关智能/设置贪睡：录屏 + 持久化行为。  
9. 权限申请流程完整：授权、拒绝、恢复授权全流程录屏。

## 8. 交付与判定

## 必交付
1. 用例执行表（已签字/日期）。  
2. iPhone + watch 录屏。  
3. iOS + watch `runtime-events.jsonl`。  
4. 问题清单（严重级别+复现步骤）。

## Go / No-Go 规则
1. 9 条验收项全部通过且有证据。  
2. 功耗满足 `<5%/小时`（至少两轮满足）。  
3. 异常场景无漏叫。  
4. 任一关键项失败则 `No-Go`。
